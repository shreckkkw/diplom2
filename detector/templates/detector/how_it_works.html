Как это работает

Описание метода для дипломной работы

1. Постановка задачи

Задача системы — обнаружить подозрительные записи в сетевом трафике без заранее заданных правил для каждого типа атаки. Вместо сигнатурного анализа используется нейросетевой автоэнкодер, который учится на примерах нормального трафика и выявляет записи, существенно отличающиеся от привычного поведения сети.

2. Признаки (features)

Каждое сетевое соединение описывается 8 числовыми признаками:

  • Количество пакетов
  • Объём переданных данных (байт)
  • Длительность соединения (сек)
  • Порт источника и назначения
  • Код протокола (TCP, UDP, HTTP...)
  • Число соединений в минуту
  • Средний размер пакета

4. Критерий аномалии

Для каждой записи вычисляется ошибка реконструкции (MSE) — среднеквадратичное отклонение восстановленных признаков от исходных. При обучении на нормальном трафике определяется порог (95-й перцентиль ошибок). Записи с ошибкой выше порога классифицируются как аномалии. Уровень риска (низкий / средний / высокий) зависит от кратности превышения порога.
